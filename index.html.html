What is init?

Ініціалізація нового репозиторію: git init

Команда git initстворює новий репозиторій Git. З її допомогою можна перетворити існуючий проект без керування версіями на репозиторій Git або ініціалізувати новий порожній репозиторій. Більшість інших команд Git неможливо використовувати без ініціалізації репозиторію, тому команда зазвичай виконується першою в рамках нового проекту.

При виконанні команди git initв поточному робочому каталозі створюється підкаталог .gitз усіма необхідними метаданими Git для нового репозиторію. Метадані включають підкаталоги для об'єктів, посилань та файлів шаблонів. Крім того, створюється файл HEAD, який вказує на поточний вилучений коміт.

Крім створення каталогу .git, кореневий каталог існуючого проекту не змінюється будь-яким чином (на відміну від SVN, Git не вимагає наявності підкаталогу .gitв кожному підкаталозі).

За замовчуванням команда git initініціалізує конфігурацію Git у підкаталозі .gitвідповідним шляхом. При необхідності шлях підкаталогу можна змінити. Якщо присвоїти змінному середовищу $GIT_DIRзначення шляху, команда git initініціалізує файли конфігурації Git по цьому шляху. Такий результат можна отримати, якщо передати команді аргумент --separate-git-dir. Часто окремий підкаталог .gitстворюється з міркувань, що системні файли конфігурації з точкою перед ім'ям ( .bashrc, .vimrcта інших.) повинні розміщуватися домашньому каталозі, а папка .git— іншому місці.
